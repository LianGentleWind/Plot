# UB-XMem 实验评估图表版式设计规划（无表格纯图阵容）

## 1. 核心制图策略：维度展平与紧凑排版
参考各类顶会（如 SIGCOMM / ISCA）的排版风格，我们将采用**多子图维度拆解**来填补全图阵容，使得版面信息极为丰富扎实：
- **单栏3行组图 (3x1 Column Subplots)**：采用单栏宽度下多行纵向堆叠，适合共享一维 X 轴的不同指标测定。
- **1行2列横向图 (1x2 Row Subplots)**：拓展单一侧重点，平行呈现两项高度互补的实验验证。
- **双栏巨阵大图 (2x3 Double-Column Matrix)**：用于最终展示系统泛化性和跨尺度适应性。

---

## 2. 实验图表全面规划 (5 组核心图)

### **Fig 1: 微观单指标多维分析 (单栏 3行1列 纵向图)**
- **X轴** (共享到底部): 数据包大小 (Message Size, 64B ~ 1MB)。
- **展示切片** (纵向上层叠三张子图，紧凑而信息密集): 
  - **(a) 通信延迟 Latency**: 证明 Load/Store 微观下限。
  - **(b) 系统吞吐 Bandwidth**: 证明 URMA 顶端吞吐能力。
  - **(c) CPU 资源开销 CPU Utilization**: 直观体现融合语义能最小化轮询和中断占用。
- *(使用 `plot_fig1_microbenchmark.py` 生成)*

### **Fig 2: 算子级验证双重审视 (跨栏 1行2列 平行图)**
- 为了丰富原本单一耗时图的信息维度，拆分出两个视角：
  - **(a) 归一化执行时间 (Normalized Execution Time)**：总耗时对比定性。
  - **(b) 网络带宽利用率 (Network Bandwidth Utilization)**：更微观地解释为何时间会有差异（例如 L/S 在大包算子中往往处于停等阻塞状态）。
- *(使用 `plot_fig2_operator.py` 生成)*

### **Fig 3: 推理阶段双极性解构 (单栏 3行1列 纵向图)**
- 将端到端推理阶段强行剥开分析：
  - **(a) Prefill 阶段通信时间**：带宽高需求区。
  - **(b) Decode 阶段通信时间**：极度延迟敏感区。
  - **(c) 总通信执行时间**：将两者的特性叠加，证明必须采用融合语义。
- *(使用 `plot_fig3_e2e_llm.py` 生成)*

### **Fig 4: 集群规模抗压性测试 (跨栏 1行2列 平行图)**
- 扩展集群多跳维度的评价角度：
  - **(a) 归一化吞吐效率 (Normalized Efficiency)**：宏观表现。
  - **(b) P99 长尾延迟 (P99 Tail Latency，对数域)**：深入追踪多跳造成的队头阻塞效应。
- *(使用 `plot_fig4_scalability.py` 生成)*

### **Fig 5: 端到端矩阵全景巨图 (双栏 2行3列 组合大图)**
- **纵向（两行）**: 集群规模 (Scale = 64 Nodes vs 256 Nodes)，测试拓扑横向放大下的抗造性。
- **横向（三列）**: 业务多样泛化性（Dense LLM, Sparse MoE, GNN/DLRM）。
- *(使用 `plot_fig5_comprehensive.py` 生成)*
